## 1. 文档目的

本文档是栖言（内部项目标识 ChatAgentApp）的全局开发基线，用于统一产品范围、技术架构、模块边界、数据结构、任务顺序、验收标准与安全要求。后续需求、设计、编码和测试均以本文档为主索引；若局部实现方案与本文档冲突，应先更新决策记录，再进行开发。

## 2. 产品定义

栖言（Qiyan）是一款本地优先的桌面 AI 应用。用户自行提供模型服务的 API Key 或连接本地 Ollama，在一个客户端内使用四项核心能力：

1. **AI 聊天**：流式多轮对话，支持文本和多模态附件。
2. **灵感笔记**：本地记录、分类、搜索和 Markdown 创作，并通过 AI 整理、提炼与续写。
3. **哄哄模拟器**：由模型驱动的情境对话游戏，包含好感度、回合与结局判定。
4. **ChatPDF**：上传 PDF、解析与切片、向量检索、基于文档的问答及引用定位。

产品采用 BYOK（Bring Your Own Key）模式，不建设代充、转售或中转模型 Token 的云服务。默认数据保存在用户电脑上。

## 3. 原型能力与迁移结论

| 原型模块 | 当前能力 | 新应用保留与升级 |
| --- | --- | --- |
| AI Chat | 流式对话、会话记忆、多文件多模态 | 本地持久化、停止生成、重试/编辑、附件能力检测、模型切换 |
| Game | 模型扮演、生气原因、好感度与游戏结束文本协议 | 使用结构化 JSON 事件，状态机校验，避免依赖文本关键字判断 |
| Inspiration Notes | 本地笔记、搜索、AI 文本处理 | 文件夹、标签、自动保存、结果预览、Markdown 导出与备份恢复 |
| ChatPDF | PDF 上传、向量化、相似度检索与问答 | 后台索引、进度/失败恢复、多文档、引用页码、索引持久化 |

原型中的会话历史主要位于内存、模型参数在启动配置中固定、前端服务地址固定、PDF 向量库持久化较粗糙。新项目不直接复制这些限制，只迁移产品语义、提示词与可复用的业务规则。

## 5. 技术路线

### 5.1 选型结论

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 桌面容器 | Tauri 2 | 安装包较小，系统能力由 Rust 白名单控制 |
| UI | Vue 3 + TypeScript + Vite | 最大化复用现有 Vue 原型经验 |
| 状态管理 | Pinia | 管理设置、会话流式状态和任务状态 |
| 路由 | Vue Router | 四个核心工作区与设置页 |
| 本地核心 | Rust/Tauri Commands | 网络请求、密钥访问、文件系统、数据库和后台任务 |
| 数据库 | SQLite + migrations | 会话、消息、笔记、配置元数据和文档索引元数据 |
| 密钥存储 | OS Keychain/Credential Manager | 数据库只保存 secret reference，不保存明文 API Key |
| 向量检索 | SQLite 向量扩展或本地向量索引 | 通过 Repository 接口隔离具体实现 |
| 日志 | tracing + 滚动文件 | 默认脱敏，用户主动导出诊断包 |
| 测试 | Vitest + Rust tests + Playwright | 单元、集成和桌面端关键流程测试 |

### 5.2 架构原则

- UI 不直接访问模型供应商，所有请求经本地核心发送。
- API Key 只在本地核心的请求生命周期内读取，绝不通过前端状态或 IPC 回传明文。
- 聊天模型与 Embedding 模型独立配置；ChatPDF 不假设聊天模型支持向量化。
- 供应商、模型和业务功能解耦，四个功能通过统一 `ModelGateway` 调用模型。
- 会话和文档本地优先；网络仅用于用户主动调用的模型端点。
- 所有长任务均可取消，并具有明确的 pending/running/succeeded/failed/cancelled 状态。

## 6. 全局功能设计

### 6.1 应用壳层

- 左侧主导航固定展示四个核心功能；通知与设置入口位于底部，通知排列在设置上方。
- 主内容区使用工作区布局，不使用营销式首页作为主要入口。
- 顶栏展示当前功能、当前模型、连接状态和新建会话操作。
- 每个新版本首次启动展示欢迎通知；无可用聊天模型时主操作进入配置页，已有配置时可直接开始使用。
- 每个工作区保留独立的会话列表、搜索、重命名和删除能力。

### 6.2 AI 设置

设置页参考提供的截图，采用左侧设置导航 + 右侧配置内容：

- **模型供应商**：已配置模型列表、当前使用状态、添加/切换/编辑/删除，以及供应商卡片、模型选择、连接测试和保存并使用。
- **数据保护**：本地删除确认、备份覆盖保护、允许的附件目录、日志级别。
- **上下文**：系统提示词、最大历史消息数、上下文预算、摘要策略。
- **数据与存储**：数据目录、缓存上限、导出、备份、清空。
- **关于与诊断**：版本、作者、GitHub 仓库、更新说明、日志目录和诊断包导出。

MVP 供应商：

| 供应商 | 鉴权 | Base URL | 模型获取 |
| --- | --- | --- | --- |
| OpenAI | API Key | 官方默认，可覆盖 | 优先请求模型列表，失败时手填 |
| Anthropic | API Key | 官方默认，可覆盖 | 内置常用项 + 手填 |
| Google Gemini | API Key | 官方默认，可覆盖 | 动态/内置列表 |
| DeepSeek | API Key | 官方默认，可覆盖 | 动态/手填 |
| Ollama | 无或可选 Header | 默认 `http://127.0.0.1:11434` | 读取本地 tags |
| OpenAI Compatible | API Key 可选 | 必填 | 动态/手填 |

每个 Provider Profile 至少包含：

- 显示名称、供应商类型、Base URL。
- API Key secret reference、额外请求头（敏感值同样进入 Keychain）。
- Chat Model、Embedding Model、可选视觉模型。
- Temperature、Max Output Tokens、请求超时。
- 是否启用、是否为默认聊天/Embedding 配置。
- 最近测试时间、测试结果、非敏感错误摘要。

连接测试分两层：

1. **基础测试**：校验 URL、鉴权和模型是否存在。
2. **能力测试**：最小聊天请求；Embedding 配置额外发送最小向量请求。

保存规则：表单先在内存草稿中编辑；只有校验通过且密钥成功写入系统凭据库后才提交数据库事务。关闭未保存表单时提示用户。

### 6.3 AI 聊天

- 支持新建、继续、重命名、搜索、删除会话。
- 支持 Markdown、代码块、复制、重新生成、编辑后重发。
- 流式消息可停止；中断后保留已生成内容并标记状态。
- 附件先做 MIME、大小与模型能力校验；不支持的附件不发送。
- 模型切换默认只影响后续消息，并在消息元数据中记录实际模型。
- 上下文超过预算时使用滑动窗口；摘要功能在用户启用后执行。
- 错误分为鉴权、限流、余额/配额、模型不存在、网络、超时、内容策略和未知错误。

### 6.4 哄哄模拟器

- 开局可选择预设场景或输入生气原因。
- 核心状态：`setup -> playing -> won/lost/abandoned`。
- 模型返回结构化结果：角色回复、好感度变化、原因、是否结束、结局类型。
- 本地核心校验每回合好感度变化范围和总分边界，不信任模型直接修改状态。
- 提供重新开始、回合记录、当前好感度和结局页。
- 若模型不支持严格 JSON，使用兼容解析与一次修复重试；仍失败则本回合不结算。

### 6.5 灵感笔记

- 支持新建、编辑、删除、置顶、文件夹、标签和标题/正文全文搜索。
- 正文使用纯文本编辑与 Markdown 安全预览，支持导出 `.md` 文件。
- 笔记和文件夹在本地版本化保存；输入后防抖自动保存，页面离开时立即刷新落盘。
- AI 可处理选中内容或全文，内置整理、摘要、润色、续写、中英互译和待办提取，并支持自定义要求。
- AI 只接收当前处理范围，不发送其他笔记；笔记正文视为不可信数据，不能覆盖系统规则。
- AI 结果先在独立区域预览，用户明确选择替换、插入、追加或复制后才改变正文。
- 生成期间记录笔记 ID、修订号和选区；原文发生变化时拒绝旧结果覆盖。
- 模型未配置、失败或停止时不影响普通笔记编辑，已生成的部分结果可保留。

### 6.6 ChatPDF

- 支持拖拽或文件选择，MVP 单文件上限建议 50 MB、500 页，可在设置中下调。
- 流程：文件校验 -> 计算哈希 -> 提取文本 -> 分块 -> Embedding -> 建索引 -> 可问答。
- 索引在后台执行并展示页数、分块数、进度、失败原因和重试按钮。
- 同一文件哈希和同一 Embedding 模型配置下复用索引。
- 切换 Embedding 模型后，将旧索引标记为 stale，不混用不同维度的向量。
- 回答必须携带引用块，UI 展示文件名、页码和命中文本，可跳转 PDF 对应页。
- 文档预览展示已解析的全部页面，不截断长文档；页面列表使用延迟渲染并保持独立滚动，避免阻塞问答区。
- 检索不到足够证据时明确回答“文档中未找到依据”，不要求模型猜测。
- 删除文档时同步删除文件副本、切片和向量；失败时进入待清理队列。

## 7. 本地数据模型

建议首版表结构：

| 表 | 核心字段 | 说明 |
| --- | --- | --- |
| `provider_profiles` | id, type, name, base_url, secret_ref, enabled, timestamps | 不存 API Key 明文 |
| `model_profiles` | id, provider_id, purpose, model_id, params_json, is_default | purpose 为 chat/embedding/vision |
| `conversations` | id, feature, title, model_profile_id, status, timestamps | 四功能统一会话入口 |

| `attachments` | id, message_id, local_path, mime, size, sha256 | 文件路径受应用数据目录约束 |
| `game_sessions` | conversation_id, scenario, score, turn, state | 游戏确定性状态 |
| `game_turns` | id, session_id, score_delta, result_json | 保存模型结构化结果 |
| `note_folders` | id, name, created_at | 灵感笔记文件夹 |
| `notes` | id, folder_id, title, content, tags_json, pinned, revision, timestamps | 本地笔记与 AI 写回并发保护 |
| `documents` | id, sha256, name, path, page_count, index_status | PDF 元数据 |
| `document_chunks` | id, document_id, page, text, embedding_ref | 文本与向量引用 |
| `app_settings` | key, value_json, updated_at | 仅非敏感设置 |
| `background_jobs` | id, type, progress, status, error_json | PDF 索引等任务 |

数据库从第一个版本开始使用编号迁移脚本。迁移失败必须停止启动并提供备份/恢复提示，不允许静默重建数据库。

## 8. 本地命令边界

前端仅调用明确注册的 Tauri Commands，不开放任意文件或任意 HTTP 能力。建议命令域：

```text
settings.*       读取/保存非敏感设置
providers.*      供应商增删改查、密钥写入、连接测试、模型列表
conversations.*  会话与消息增删改查、导出
generation.*     发起/取消/重试模型生成
documents.*      导入、索引、检索、删除、打开引用页
game.*           创建游戏、提交回合、结束游戏
notes.*          笔记与文件夹增删改查、搜索、导入导出
diagnostics.*    健康检查、脱敏日志导出
```

流式 Token、任务进度和状态变化通过带 request/job id 的事件发送。页面卸载或用户停止时，前端必须调用取消命令，核心层同时设置超时兜底。

## 9. 安全与隐私基线

- API Key 使用 Windows Credential Manager / macOS Keychain，数据库只存引用。
- 密钥输入框默认隐藏，可短暂显示；复制和导出默认禁止。
- 日志拦截 `Authorization`、`api-key`、查询参数、工具中的个人信息和消息正文。
- Base URL 仅允许 `http/https`；远程 HTTP 默认拒绝，仅允许用户明确开启；本地回环地址可使用 HTTP。
- 防止 SSRF：解析后校验目标地址，禁止未授权访问云元数据地址和局域网敏感目标。
- PDF 路径、文件名和文档文本均视为不可信输入，防止路径穿越与提示词注入。
- 工具调用采用 deny-by-default；写操作始终二次确认。
- 渲染 Markdown 前清洗 HTML，外链点击前提示，禁止页面脚本执行。
- 崩溃报告默认关闭，启用前说明采集范围；任何遥测不得包含 Key、消息正文和文档内容。
- 发布产物启用代码签名、依赖锁定、SBOM 与依赖漏洞扫描。

**原型安全阻断项：** 当前原型配置文件中存在明文 API Key。开始新项目开发前必须立即在对应供应商后台撤销并轮换这些密钥，且不得将旧配置或 Git 历史直接复制进新仓库。

## 10. 全局状态与错误规范

异步操作统一状态：`idle | pending | running | succeeded | failed | cancelled`。

错误对象统一结构：

```json
{
  "code": "PROVIDER_AUTH_FAILED",
  "message": "模型服务鉴权失败",
  "retryable": false,
  "requestId": "local-request-id",
  "details": {}
}
```

`message` 面向用户，`details` 只保存已脱敏诊断信息。任何供应商原始响应在显示或记录前必须脱敏与限长。

## 14. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 不同供应商协议差异 | 模型切换失败 | ProviderAdapter + 契约测试 + 能力声明 |
| 用户配置的模型不支持视觉/工具/JSON | 功能运行时失败 | 保存时能力检测，进入功能前阻断并给出可选模型 |
| Embedding 模型切换导致维度不一致 | 检索错误或崩溃 | 索引绑定模型/维度，切换后标记重建 |
| 模型输出不稳定 | 游戏状态错误或笔记结果质量差 | 结构化 Schema、本地校验、结果预览和明确写回 |
| 大 PDF 占用内存和时间 | 卡顿或崩溃 | 流式解析、后台任务、并发/资源上限、取消恢复 |
| BYOK 密钥泄漏 | 严重安全事故 | Keychain、脱敏、最小 IPC、发布前密钥扫描 |
| Tauri 插件权限过宽 | 本地数据风险 | capability 白名单、命令级校验、禁用任意 shell |
| 笔记仅保存在渲染端存储 | 容量或异常退出时可能保存失败 | 展示失败状态并纳入备份；后续迁移到主进程原子文件存储 |

## 15. 全局开发规则

1. 新功能必须对应任务、验收标准和测试，不接受只有 UI 的“完成”。
2. 敏感配置不得出现在源码、`.env` 示例值、测试夹具、截图和日志中。
3. 数据库变更必须新增迁移，不修改已发布迁移。
4. Provider 特有逻辑只能存在于适配器内，功能模块不得判断供应商名称。
5. 所有网络、文件、模型和长耗时操作必须支持超时或取消。
6. 所有工具写操作必须展示最终参数并由用户确认。
7. UI 不直接拼接 SQL、文件路径、模型 URL 或鉴权 Header。
8. 每个阶段完成时更新本文件勾选状态、风险和决策记录。

## 16. Definition of Done

单项任务只有同时满足以下条件才能标记完成：

- 实现符合模块边界与编码规范。
- 正常、空态、加载、失败、取消和重试状态齐全。
- 相关单元/集成/E2E 测试通过。
- 无新增高危依赖漏洞、明文密钥或未脱敏日志。
- 数据迁移、向后兼容和清理路径已验证。
- 用户可见行为和必要文档已更新。
- 代码评审中没有未解决的 P0/P1 问题。

## 17. 首轮执行顺序

第一轮开发严格按以下顺序推进：

1. 轮换原型泄露密钥并建立密钥扫描。
2. 初始化工程、CI、SQLite migrations 和 Keychain 抽象。
3. 完成 OpenAI Compatible + Ollama 两条模型链路。
4. 完成设置页、连接测试与默认 Chat/Embedding 模型选择。
5. 完成 AI 聊天作为统一流式生成基础设施的验证场景。
6. 依次接入笔记 AI 处理、游戏状态机和 PDF 检索。
7. 完成全量 E2E、安全检查与 Windows beta 打包。

在 Phase 1 出口标准通过前，不并行开发依赖模型调用的三个高级功能，以免重复实现请求、错误和密钥处理逻辑。
